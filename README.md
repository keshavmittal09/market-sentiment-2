# Trader Sentiment Analysis

This repository contains the analysis for the Primetrade.ai Data Science Intern Assignment. The objective of this project is to analyze the relationship between market sentiment (Bitcoin Fear/Greed) and trader behavior/performance on Hyperliquid.

## Files
- `sentiment.csv`: Bitcoin Fear and Greed index dataset (download separately)
- `trader_data.csv`: Historical trader dataset from Hyperliquid (download separately)
- `analysis.ipynb`: Main Jupyter Notebook containing all data processing and exploratory analysis.
- `charts/`: Folder containing the output visualizations generated by the notebook.

## Setup and Execution
1. **Clone this repository**: 
   ```bash
   git clone https://github.com/keshavmittal09/market-sentiment-2.git
   cd market-sentiment-2
   ```
2. **Download Datasets**: Place the downloaded `sentiment.csv` and `trader_data.csv` files in the root directory.
3. **Install Dependencies**: Make sure you have pandas, matplotlib, and jupyter installed.
   ```bash
   pip install pandas matplotlib jupyter
   ```
4. **Run the Notebook**: Open Jupyter and run all cells in `analysis.ipynb`.
   ```bash
   jupyter notebook analysis.ipynb
   ```

## Methodology (Part A & B)
1. **Data Cleaning & Alignment**:
   - Dates from both files were normalized and parsed identically to align data daily.
   - Merged the daily sentiment with the trades data on the `date` key.
2. **Metrics Engineered**:
   - Calculated daily Profit & Loss (PnL), win rate, and total trade volume per account.
   - Engineered an aggregate long/short ratio to monitor market directional bias per day.
3. **Exploratory Analysis**:
   - Grouped the performance dataset by the `sentiment_class` (e.g., Fear, Greed, Extreme Fear) to compare variations in trader success and behavior.
   - Segmented traders by activity (Frequent vs Infrequent) and trade size (Small vs Large) using quantiles to understand how different styles perform under distinct market psychologies.

## Insights Discovered
*Note: A complete output with exact figures is printed into the console upon running the script, and charts are generated in the `/charts` directory.*

1. **Win Rates vs Sentiment**: The analysis generally shows win rates differ between Fear and Greed regimes, signifying that when market emotion changes, existing strategies' edge often shifts.
2. **Activity Correlation**: Some trader segments (such as the most frequent traders) may exhibit heightened activity and larger trade sizes during "Greed", which occasionally corresponds to diminished PnL if they are caught in reversals.
3. **Segment Differences**: High-frequency traders often behave differently compared to lower-frequency, structural traders during "Extreme Fear" days, heavily influencing their variance in daily PnL.

## Strategy Recommendations (Part C)
Based on the observations:
1. **Regime-based Sizing Adjustments**: During "Extreme Fear" or highly uncertain periods where win rates tend to systematically drop or become highly volatile, algorithms should actively scale down their size (or reduce leverage) until the regime normalizes back into standard Fear/Greed parameters.
2. **Frequency Filtering**: The data identifies diminished returns for overly frequent trading during certain sentiment regimes. A rule of thumb should be: require a higher-conviction signal or tighter technical confirmation to take a trade when sentiment is extremely polarized.
